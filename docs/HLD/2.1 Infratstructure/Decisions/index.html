1. Application Runtime‚Äã

GCP Cloud Run vs AWS App Runner / Fargate vs Azure Container Apps‚Äã

‚òÅ 1. GCP Cloud Run ‚Äî Always Free Tier‚Äã

  • 2 million requests/month free
  • 360,000 GB-seconds and 180,000 vCPU-seconds free per month
  • 1 GB egress to most regions free
  • No need to rewrite Spring Boot for GCP ‚Äî you just package as a Docker image and deploy.

Why minimal code changes?

  • Cloud Run runs any container that listens on a port.

  • For Spring Boot, you just:

    • Expose server.port=${PORT:8080} in application.properties
    • Package with Dockerfile
    • Deploy via gcloud run deploy

‚òÅ 2. AWS Equivalent ‚Äî AWS Lambda + API Gateway (Always Free)‚Äã

AWS doesn‚Äôt have an always free container hosting exactly like Cloud Run. Closest options:

ServiceAlways FreeNotes
AWS Lambda1M requests/month + 400,000 GB-secEvent-based, can run Java but cold starts hurt
AWS App Runner‚ùå (no always free)Container hosting like Cloud Run, but only a 12-month free trial
AWS Fargate (ECS)‚ùå (no always free)Pay per vCPU-second and GB-second

For AWS, Lambda is free forever but have to adapt Spring Boot app into a Lambda-friendly format (using AWS Serverless Java Container or Quarkus/Spring Native).

‚òÅ 3. Azure Equivalent ‚Äî Azure Functions + Container Apps‚Äã

Azure also doesn‚Äôt have a direct Cloud Run‚Äìstyle always free container runtime.

ServiceAlways FreeNotes
Azure Functions1M requests/month + 400,000 GB-secEvent-based, similar to Lambda
Azure Container Apps180,000 vCPU-sec + 360,000 GiB-sec free per monthClosest to Cloud Run
Azure App Service1 GB storage, 60 CPU-min/dayBasic tier, supports Java but limited

Azure Container Apps is the closest match to Cloud Run‚Äôs pricing and model, and its Always Free tier is lifetime.

2. Relational Data Storage‚Äã

‚òÅ 1. Google Cloud (GCP)‚Äã

Cloud SQL ‚Üí ‚ùå No lifetime free tier Only included in the 12-month $300 free trial. After that, it‚Äôs fully paid.

But: They do have AlloyDB Free Trial & Spanner Emulator for dev, but not lifetime hosted SQL.

Always Free DB option ‚Üí Cloud Firestore (NoSQL) and Datastore Mode have a lifetime free tier, but not traditional SQL.

‚òÅ 2. AWS‚Äã

Amazon RDS ‚Üí ‚ùå No lifetime free tier You get 750 hours/month of db.t2.micro (or t3.micro) for 12 months only.

Always Free DB option ‚Üí

  • Amazon Aurora Serverless v2 ‚Üí No lifetime free, but pay-per-use (low usage can be cheap).
  • Amazon DynamoDB (NoSQL) ‚Üí 25 GB storage lifetime free.

‚òÅ 3. Azure‚Äã

Azure Database for MySQL/PostgreSQL/SQL Server ‚Üí ‚ùå No lifetime free tier Only included in the 12-month free trial.

Always Free DB option ‚Üí

  • Azure Cosmos DB (NoSQL) ‚Üí 1,000 RU/s + 25 GB lifetime free.

ProviderSQL ServiceLifetime Free?Notes

üí° Workarounds for Lifetime Free SQL in Cloud‚Äã

Since none of the big 3 offer hosted SQL lifetime free:

  1. Run SQL DB in a free container runtime:

    • GCP Cloud Run / Cloud Run Jobs (Always Free)
    • Azure Container Apps (Always Free)
    • AWS Lambda + RDS Proxy (not free lifetime, but low-cost)
    • Free-tier VPS (Oracle Cloud Free Tier, always free 2x AMD instances ‚Üí you can run MySQL/Postgres yourself)

  2. Use free-tier DB hosting providers:

    • Neon.tech ‚Üí Postgres, 3 GB storage, free forever
    • Supabase ‚Üí Postgres, 500 MB storage, free forever
    • Railway.app ‚Üí Postgres/MySQL, limited hours/month free
    • Planetscale ‚Üí MySQL-compatible, free 5 GB storage
