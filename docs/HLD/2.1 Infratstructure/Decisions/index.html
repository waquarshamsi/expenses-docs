Infrastructure Decisions

1. Application Runtime​

GCP Cloud Run vs AWS App Runner / Fargate vs Azure Container Apps​

☁ 1. GCP Cloud Run — Always Free Tier​

  • 2 million requests/month free
  • 360,000 GB-seconds and 180,000 vCPU-seconds free per month
  • 1 GB egress to most regions free
  • No need to rewrite Spring Boot for GCP — you just package as a Docker image and deploy.

Why minimal code changes?

  • Cloud Run runs any container that listens on a port.

  • For Spring Boot, you just:

    • Expose server.port=${PORT:8080} in application.properties
    • Package with Dockerfile
    • Deploy via gcloud run deploy

☁ 2. AWS Equivalent — AWS Lambda + API Gateway (Always Free)​

AWS doesn’t have an always free container hosting exactly like Cloud Run. Closest options:

ServiceAlways FreeNotes
AWS Lambda1M requests/month + 400,000 GB-secEvent-based, can run Java but cold starts hurt
AWS App Runner❌ (no always free)Container hosting like Cloud Run, but only a 12-month free trial
AWS Fargate (ECS)❌ (no always free)Pay per vCPU-second and GB-second

For AWS, Lambda is free forever but have to adapt Spring Boot app into a Lambda-friendly format (using AWS Serverless Java Container or Quarkus/Spring Native).

☁ 3. Azure Equivalent — Azure Functions + Container Apps​

Azure also doesn’t have a direct Cloud Run–style always free container runtime.

ServiceAlways FreeNotes
Azure Functions1M requests/month + 400,000 GB-secEvent-based, similar to Lambda
Azure Container Apps180,000 vCPU-sec + 360,000 GiB-sec free per monthClosest to Cloud Run
Azure App Service1 GB storage, 60 CPU-min/dayBasic tier, supports Java but limited

Azure Container Apps is the closest match to Cloud Run’s pricing and model, and its Always Free tier is lifetime.

📊 Summary Table​

ProviderAlways Free Container HostingMonthly Free AllowanceMinimal Code Changes for Spring Boot?
GCPCloud Run2M req + 360k GB-sec + 180k vCPU-sec✅ Yes
AWSLambda (no container always free)1M req + 400k GB-sec⚠ Needs Lambda adaptation
AzureContainer Apps180k vCPU-sec + 360k GiB-sec✅ Yes

💡 If the goal is just run my Spring Boot REST API with no vendor lock-in code, Cloud Run and Azure Container Apps are the most hassle-free lifetime free options.

1️⃣ Cloudflare Workers + Cloudflare Workers AI + Durable Objects​

  • Always Free: 100k requests/day free
  • Supports Java via Cloudflare Workers with WASM or HTTP API (but running Spring Boot is not practical directly here — better for smaller Java apps or compiled-to-native).
  • Not ideal for full JVM-based Spring Boot (better for lightweight Java or Kotlin compiled to native).

2️⃣ Fly.io​

  • Always Free Tier:

    • 3 shared CPUs
    • 256 MB RAM
    • 3 GB persistent storage
    • 160 GB outbound traffic/month

  • Runs Docker containers directly — works with a standard Spring Boot image.

  • Minimal vendor lock-in — similar to Cloud Run.

3️⃣ Railway.app​

  • Free Tier:

    • 500 hours/month runtime
    • 1 GB RAM
    • 1 GB persistent disk

  • Supports Dockerfile deploys.

  • Can run a plain Spring Boot container without changes.

  • Can also host PostgreSQL/MySQL free (500 MB DB).

4️⃣ Render​

  • Free Web Services:

    • 750 hours/month (enough for always-on single service)
    • 512 MB RAM
    • 750 build minutes/month

  • Runs Docker containers or directly builds from Maven/Gradle.

  • SSL + custom domains free.

5️⃣ Oracle Cloud Always Free​

  • Lifetime free:

    • 2x AMD compute instances (1 OCPU, 1 GB RAM each)
    • 200 GB block storage
    • Load balancer

  • Can run Docker + Spring Boot directly on the free VM.

  • Essentially a free VPS.

6️⃣ Koyeb​

  • Free Tier:

    • 1 shared CPU
    • 512 MB RAM
    • 1 GB storage
    • 100 GB outbound traffic/month

  • Simple container hosting.

  • Can deploy Docker images or connect to GitHub repo.

📊 Quick Comparison: Always Free Hosting for Containers​

ProviderFree CPU/RAMAlways Free?Supports Docker?Good for Spring Boot?
GCP Cloud Run0.18 vCPU / 512MB✅ Lifetime✅✅
Azure Container Apps0.25 vCPU / 512MB✅ Lifetime✅✅

💡 For your use case (Spring Boot REST API + S3-compatible storage like MinIO/Cloudflare R2), the easiest free hosting paths are:

  • Cloud Run → Easiest to deploy, fully managed.
  • Railway.app → Includes free Postgres.
  • Oracle Cloud Always Free → More control, can run DB + API in same VM.
  • Render → Simple deploy, free SSL, GitHub auto-deploy.

Conclusion​

DecisionID #4: Cloud Run is the most hassle-free lifetime free option for running a Spring Boot REST API. Hence will use Google Cloud Run for hosting the microservices.

2. Relational Data Storage​

☁ 1. Google Cloud (GCP)​

Cloud SQL → ❌ No lifetime free tier Only included in the 12-month $300 free trial. After that, it’s fully paid.

But: They do have AlloyDB Free Trial & Spanner Emulator for dev, but not lifetime hosted SQL.

Always Free DB option → Cloud Firestore (NoSQL) and Datastore Mode have a lifetime free tier, but not traditional SQL.

☁ 2. AWS​

Amazon RDS → ❌ No lifetime free tier You get 750 hours/month of db.t2.micro (or t3.micro) for 12 months only.

Always Free DB option →

  • Amazon Aurora Serverless v2 → No lifetime free, but pay-per-use (low usage can be cheap).
  • Amazon DynamoDB (NoSQL) → 25 GB storage lifetime free.

☁ 3. Azure​

Azure Database for MySQL/PostgreSQL/SQL Server → ❌ No lifetime free tier Only included in the 12-month free trial.

Always Free DB option →

  • Azure Cosmos DB (NoSQL) → 1,000 RU/s + 25 GB lifetime free.

📊 Comparison Table: Always Free SQL Hosting​

ProviderSQL ServiceLifetime Free?Notes
GCPCloud SQL❌Only paid after 12 months
AWSRDS❌Free only for 12 months
AzureAzure SQL❌Free only for 12 months

💡 Workarounds for Lifetime Free SQL in Cloud​

Since none of the big 3 offer hosted SQL lifetime free:

  1. Run SQL DB in a free container runtime:

    • GCP Cloud Run / Cloud Run Jobs (Always Free)
    • Azure Container Apps (Always Free)
    • AWS Lambda + RDS Proxy (not free lifetime, but low-cost)
    • Free-tier VPS (Oracle Cloud Free Tier, always free 2x AMD instances → you can run MySQL/Postgres yourself)

  2. Use free-tier DB hosting providers:

    • Neon.tech → Postgres, 3 GB storage, free forever
    • Supabase → Postgres, 500 MB storage, free forever
    • Railway.app → Postgres/MySQL, limited hours/month free
    • Planetscale → MySQL-compatible, free 5 GB storage
